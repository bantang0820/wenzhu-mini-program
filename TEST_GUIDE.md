# API 测试脚本使用说明

## 📝 简介

`test-api.js` 是一个全面的 API 接口测试脚本，可以自动测试所有后端接口，模拟真实用户的操作流程。

## 🚀 快速开始

### 1. 确保后端服务运行

后端服务必须运行在：`https://1dt2po0565100.vicp.fun`

### 2. 运行测试脚本

```bash
# 在项目根目录运行
node test-api.js
```

## ⚠️ 注意事项

### 1. 微信登录测试
- 使用的是模拟 code，不是真实的微信 code
- 真实环境需要在微信小程序中获取有效 code
- 测试可能返回测试用户数据

### 2. 依赖关系
- 某些测试依赖前面的测试结果
- 例如：测试7（兑换会员码）依赖测试4（登录）的 token
- 如果前面的测试失败，后续相关测试会被跳过

### 3. 网络要求
- 确保能够访问 `https://1dt2po0565100.vicp.fun`
- 如果使用本地服务器，需要修改 `API_BASE_URL`

### 4. 数据库状态
- 测试会实际操作数据库
- 兑换码测试会消耗兑换码
- 建议使用专门的测试环境

## 🛠️ 自定义配置

### 修改服务器地址

编辑 `test-api.js` 中的配置：

```javascript
// 修改 API 地址
const API_BASE_URL = 'http://localhost:3000/api'; // 本地开发
const USE_HTTPS = false; // 本地 HTTP
```

### 修改测试兑换码

在 `testRedeemCode()` 函数中修改：

```javascript
const testCode = 'YOUR_CODE'; // 使用你的兑换码
```

## 🐛 故障排除

### 问题1: 连接失败
```
请求失败: connect ECONNREFUSED
```
**解决**: 确保后端服务正在运行

### 问题2: 所有测试失败
**解决**:
1. 检查 API 地址是否正确
2. 检查后端服务是否正常
3. 查看后端日志

### 问题3: 登录测试失败
**解决**:
- 这是正常的，因为使用模拟 code
- 后端可能返回错误或测试数据
- 不影响其他接口测试

### 问题4: 兑换码失败
**解决**:
- 确保数据库中有该兑换码
- 检查兑换码是否已被使用
- 尝试使用其他测试兑换码
